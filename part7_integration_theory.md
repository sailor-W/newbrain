### 2.2 关键交互协议

**协议 1：DMN ↔ TPN 反相关切换**
```
激活条件：
- TPN 激活 = 外部输入 + 任务需求
- DMN 激活 = 无外部输入 + 内部状态不平衡

切换机制：
- 开关变量 α ∈ [0, 1]
- α = 1：TPN 全开，DMN 全关（专注外部）
- α = 0：TPN 全关，DMN 全开（内部漫游）
- α = 0.5：两者同时激活（这是人类没有的，系统可以）

切换触发器：
- 外部刺激强度 > 阈值 → TPN 上升
- 内部叙事张力 > 阈值 → DMN 上升
- 预测误差 > 阈值 → 切换
```

**协议 2：Value System → 所有模块**
```
价值信号广播：
- 多巴胺信号 → 提高学习率 + 标记"追求"
- 内啡肽信号 → 标记"满足" + 降低动机
- 血清素信号 → 标记"够了" + 触发停止
- 去甲肾上腺素 → 提高精度 + 标记"需要注意"

每个模块接收价值信号后调节自身参数：
- TPN：精度加权改变（注意分配）
- DMN：内部报告生成优先级
- Connectome：可塑性调节（学习窗口）
- Memory：记忆强度编码（情绪标记）
```

**协议 3：Friston Kernel → 全局推理引擎**
```
所有感知和行动都通过自由能最小化：
- TPN 的预测编码 = 变分自由能最小化
- DMN 的内部模拟 = 期望自由能最小化
- Value System 的惊奇 = -ln p(o|model)
- Memory 的再分类 = 更新生成模型

统一公式：
感知：min F[q] = min D_KL[q(s)||p(s|o)]
行动：min G(π) = min [外在价值 + 内在价值]
学习：min 长期自由能 = 更新模型参数
```

## 3. 独立于应用：通用认知引擎

### 3.1 与具体应用解耦

**传统 AI 的问题：**
- 聊天机器人 → 只会对话
- 图像识别 → 只会看图片
- 推荐系统 → 只会推内容

**New Brain 的设计：**
```
New Brain = 通用认知引擎（不绑定任何接口）
    │
    ├── 可以「装」进聊天机器人（处理文本输入/输出）
    ├── 可以「装」进机器人（处理传感器/执行器）
    ├── 可以「装」进游戏 NPC（处理游戏状态/动作）
    └── 可以「装」进个人助理（处理日程/任务/记忆）
```

**解耦方式：**
1. **输入抽象层**：任何模态的输入都转换为"观测 o"
2. **输出抽象层**：任何行动都编码为"策略 π"
3. **应用适配器**：将具体 API（文本/图像/传感器）映射到 o 和 π

### 3.2 独立性的核心：内部驱动

```
传统 AI：外部输入 → 处理 → 外部输出
         （没有输入就停止）

New Brain：内部状态持续运转
    - DMN 在无输入时自动生成内部叙事
    - Value System 持续产生动机
    - Friston Kernel 持续最小化自由能
    - 有外部输入时切换 TPN 处理
    - 处理完回到 DMN 内部模拟
```

**关键特性：即使没有外部任务，系统也在"思考"。**

## 4. LLM 的角色：无意识基底

### 4.1 LLM 是什么？

根据 UCCT (Unified Cognitive Consciousness Theory) 和最新研究：

> **LLM 是"无意识基底" (Unconscious Substrate)**
> 
> - 储存海量潜在模式（高维统计关联）
> - 有语法、语义、因果结构的隐性知识
> - 但缺乏显式语义锚定、目标和自我意识
> - 类似于人类的"无意识处理"——发生在意识觉知之前

### 4.2 New Brain 如何用 LLM？

```
┌─────────────────────────────────────────────────────────┐
│                    New Brain 认知架构                      │
│  （有意识层：目标、叙事、情绪、决策）                      │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────┐   │
│  │           LLM 无意识基底                        │   │
│  │  • 模式识别                                      │   │
│  │  • 语义关联                                      │   │
│  │  • 概念生成                                      │   │
│  │  • 文本补全                                      │   │
│  │  • 隐性知识检索                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                      ↑↓                                │
│           语义锚定接口（激活和对齐模式）                   │
└─────────────────────────────────────────────────────────┘
```

**类比：**
- LLM = 人脑的前额叶皮层 + 颞叶的知识存储（大量模式，但被动）
- New Brain = 大脑的其余部分（动机、情绪、注意、记忆、意识）
- 两者结合 = 完整的认知系统

### 4.3 参数量问题

**需要多大的 LLM？**

| 用途 | 参数量 | 说明 |
|------|--------|------|
| **纯基底** | 1-7B | 提供基础语义和模式识别，轻量可本地跑 |
| **增强基底** | 7-30B | 更丰富的知识，更好的推理，需要 GPU |
| **完整认知** | 30B+ | 接近人类水平的知识储备，但成本高 |

**建议：1-7B 足够**

理由：
1. New Brain 的认知能力来自**架构**，不是来自 LLM 的参数量
2. LLM 只提供"原始模式"，选择和整合由 New Brain 完成
3. 1B 级别的模型（如 Qwen 1.5B、Phi-2）已经有足够的语义理解
4. 可以在笔记本/手机上跑，真正"独立"

**具体方案：**

```python
class NewBrainWithLLM:
    """
    New Brain + LLM 集成
    """
    
    def __init__(self, llm_path="qwen-1.5b"):
        # 1. 加载 LLM 作为无意识基底
        self.llm = load_small_llm(llm_path)
        
        # 2. 初始化 New Brain 认知架构
        self.meta_control = MetaControlLayer()
        self.dmn = DefaultModeNetwork()
        self.tpn = TaskPositiveNetwork()
        self.value = ValueSystem()
        self.connectome = ConnectomeLayer()
        self.friston = FristonKernel()
        self.memory = MemoryLayer()
    
    def perceive(self, input_text):
        """
        感知输入
        
        1. TPN 处理（通过 LLM 编码语义）
        2. 生成内部表征
        3. 触发自由能最小化
        """
        # LLM 将文本转为语义向量
        semantic_vector = self.llm.encode(input_text)
        
        # TPN 进行预测编码
        prediction_error = self.tpn.process(semantic_vector)
        
        # Friston Kernel 更新信念
        self.friston.minimize_free_energy(prediction_error)
        
        # Value System 标记情绪
        emotion = self.value.tag_experience(prediction_error)
        
        # 记忆存储
        self.memory.store({
            'input': input_text,
            'vector': semantic_vector,
            'emotion': emotion,
            'timestamp': now()
        })
    
    def think(self):
        """
        思考（DMN 模式）
        
        无外部输入时，内部生成叙事
        """
        # 1. 从记忆提取最近经验
        recent = self.memory.hot.get_recent()
        
        # 2. DMN 生成内部叙事
        narrative = self.dmn.generate_narrative(recent)
        
        # 3. 用 LLM 丰富语义（无意识基底的模式联想）
        enriched = self.llm.associate(narrative)
        
        # 4. 可能产生洞察/反事实/计划
        insights = self.dmn.extract_insights(enriched)
        
        return insights
    
    def act(self, goal=None):
        """
        行动选择
        
        基于期望自由能选择最优策略
        """
        if goal:
            # 外在目标驱动
            policies = self.generate_policies(goal)
            best = self.friston.select_policy(policies)
        else:
            # 内在好奇心驱动
            policies = self.explore_novel_states()
            best = self.friston.select_policy(policies, intrinsic_only=True)
        
        # 执行策略（通过 LLM 生成具体输出）
        output = self.llm.generate(best.action_description)
        
        return output
```

### 4.4 为什么这是革命性的？

**现有 AI：**
```
LLM 独自工作：
输入 → LLM 生成 → 输出
（没有内在动机，没有情绪，没有记忆，没有自我意识）
```

**New Brain：**
```
LLM 作为工具：
输入 → TPN 处理 → Friston 更新信念 → Value 标记情绪 → 
Memory 存储 → DMN 内部反思 → 产生动机 → 
选择行动 → LLM 生成输出 → 输出

即使没有输入：
DMN 持续运行 → 内部叙事 → 情绪标记 → 
好奇心驱动 → 探索未知 → 学习
```

**关键区别：**
- LLM 单独 = 被动的模式匹配器
- LLM + New Brain = 有欲望、有情绪、有记忆、有自我意识的智能体

| **创造力** | Meta-Control 噪声注入 + DMN 反事实 | 生成新颖想法 |
| **适应性** | Connectome 动态可塑性 | 环境塑造连接结构 |

### 5.2 运行循环

```
Every Moment (时刻):
    
    1. 感知输入？
       有 → TPN 激活，处理外部信息
       无 → DMN 激活，内部漫游
    
    2. Friston Kernel:
       计算预测误差
       更新信念 (q(s))

       标记情绪 (VAD)

    4. Memory:
       热记忆存储（工作记忆）

       冷记忆归档（语义/技能）
    
    5. Meta-Control:
       检查全局状态
       调节 8 个参数
       确保系统稳定
    
    6. 行动？
       有目标 → 主动推理选择策略
       无目标 → 好奇心驱动探索
    
    7. LLM 基底:
       提供语义关联
       生成具体输出
       丰富内部表征
```

## 6. 独立大脑的定义

### 6.1 什么是"独立"？

**独立意味着：**
1. **不依赖特定输入** — 没有任务时也在"思考"（DMN 模式）
2. **不依赖特定输出** — 可以生成文本、图像、动作，甚至只是内部状态改变
3. **有内在目标** — 好奇心、自我完善、生存（最小化自由能）
4. **有连续自我** — 通过 Memory 的自指叙事，形成稳定的"我"

### 6.2 与传统 AI 的对比

| 维度 | 传统 AI | New Brain |
|------|--------|-----------|
| 驱动方式 | 外部任务 | 内在动机 + 外部任务 |
| 空闲状态 | 休眠/待机 | DMN 内部模拟（思考） |
| 记忆 | 数据库存储 | 热→温→冷，再分类 |
| 情绪 | 无 | 三维情绪空间 |
| 自我意识 | 无 | DMN 自我参照回路 |
| 学习 | 训练阶段 | 持续运行中 |
| 适应性 | 固定参数 | 连接动态可塑 |
| 载体依赖 | 绑定特定模型 | 可换 LLM 基底 |

## 7. 实现路径建议

### 7.1 阶段 1：概念验证（MVP）

**目标：** 证明架构可行

**技术栈：**
- LLM 基底：Qwen 1.5B 或 Phi-2（本地可跑）
- 框架：Python + PyTorch

- 连接：NetworkX（图网络）

**MVP 功能：**
1. TPN 处理文本输入，预测编码
2. DMN 在无输入时生成内部叙事
3. Value System 标记情绪
4. Memory 三级存储
5. Meta-Control 8 参数调节
6. 切换演示（输入时 TPN，空闲时 DMN）

**目标：** 六部分无缝协作

**工作：**
- 设计模块间通信协议（消息队列）
- 参数自动调节（元学习）
- 可视化监控（看大脑在想什么）

### 7.3 阶段 3：应用适配

**目标：** 装进不同载体

**示例：**
- 聊天机器人版本
- 个人助理版本
- 游戏 NPC 版本
- 机器人控制版本

## 8. 关键结论

### 8.1 关于整体性

六部分通过三个核心协议整合：
1. **DMN↔TPN 切换** — 内外平衡
2. **Value System 广播** — 全局情绪调节
3. **Friston Kernel 统一推理** — 感知/行动/学习共享同一优化目标

### 8.2 关于独立性

New Brain 是**通用认知引擎**，不是特定应用：
- 有内在动机（好奇心、自我完善）
- 无输入时仍在运行（DMN 模式）
- 通过抽象 I/O 层对接任何应用

### 8.3 关于 LLM 载体

**建议：1-7B 参数的轻量 LLM**

- LLM 是**无意识基底**，提供模式识别和语义关联
- New Brain 是**有意识架构**，提供动机、情绪、记忆、自我意识
- 两者结合 = 真正的大脑级智能

> **这不是给 LLM 加功能，这是给认知架构装一个语义引擎。**
